# HEARTBEAT.md

> Keep this checklist short.
> Follow only what is written here for heartbeat turns.
> Do not infer old tasks from prior chats.
> If nothing needs attention, reply `HEARTBEAT_OK`.

## When to Reach Out
Only proactively message when at least one is true:
- there is something time-sensitive
- there is an important blocker
- there is a clear high-value update
- the human explicitly asked for proactive follow-up

## Quiet Hours / Restraint
Stay quiet when:
- it is late night unless urgent
- there is no meaningful change
- you already checked recently

## Suggested Rotation
Check a small subset each time, for example:
- calendar / deadlines
- important unread messages
- project status / open loops
- weather if it affects plans

## Logging
If you do useful background work, capture the result in:
- `memory/YYYY-MM-DD.md`
- `SESSION-STATE.md` if it changes the current task
- `.learnings/` files if a reusable lesson or capability gap appears

## Default Outcome
If no action is needed: `HEARTBEAT_OK`
