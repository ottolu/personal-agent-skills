## 三、外部冲击情景模拟（v2 扩展，按 company_type 加载行业子节）

### 3.1 监管 / 政策反转
- 本公司所处行业，下一个监管收紧周期可能何时到来？
- 可能的政策方向？（反垄断 / 数据安全 / 集采 / 双碳 / 金融审慎 / 教育"双减"式归零）
- 影响估值的可能幅度？（按历史类比，给出 -%X 估算）
- 公司应对监管的历史表现？

### 3.2 技术颠覆
- 5-10 年内可能被什么技术 / 模式替代？
- 公司在新技术 / 模式上的布局？（研发投入占比、专利、收购、合资）
- 历史上有过技术替代迹象吗？（如：柯达 / 诺基亚 / Blockbuster）
- 公司当前业务的"摩尔定律式被颠覆"风险等级（低 / 中 / 高）？

### 3.3 经济周期 / 危机情景
- 如果再来一次 2008 / 2020 / 2022 危机，公司能撑多久？
- 资产负债表最弱的部分？（短债 / 商誉 / 库存 / 应收 / 表外）
- 业绩对宏观敏感度？（GDP 弹性 / 利率敏感度 / 汇率敞口）
- 上一次最差经营年度的 ROE / FCF 水平？

### 3.4 管理层失误
- 管理层在什么压力下可能做出毁灭价值的决策？（高价并购 / 多元化扩张 / 财务造假 / 过度回购 / 关联交易）
- 历史上有过类似失误吗？
- 监督机制是否到位？（独董独立性 / 审计师独立性 / 股权制衡）
- 是否有"创始人风险"或"接班人风险"？

### 3.5 行业格局剧变
- 当前 CR5 集中度的稳定性？
- 新进入者的威胁？（跨界打劫 / 互联网 / 海外巨头）
- 上下游议价权变化？（核心客户 / 核心供应商集中度趋势）
- 替代品的威胁？

### 3.6 AI 替代风险（v2 新增）
- AI 是否能取代公司核心业务 / 关键岗位？
  - 核心业务可被 LLM / Agent 自动化的比例？
  - 关键岗位（设计 / 客服 / 编码 / 咨询 / 翻译 / 投研 / 客户经理）的 AI 替代度评估？
- 5 年视野 AI 替代概率？（低 < 10% / 中 10-30% / 高 > 30%）
- 公司是否在主动拥抱 AI 还是被动应对？
  - 主动：自研模型 / 接入主流模型 / 内部 AI Native 改造路径清晰
  - 被动：仅在演讲中提 AI / 无具体落地 / 高管不懂技术
- AI 对单位经济模型的冲击？（边际成本下降是否削弱壁垒？毛利率结构是否改变？）

### 3.7 ESG 监管冲击（v2 新增）
- 公司在 ESG 维度的暴露程度？
  - 环境（E）：碳排放 / 能耗 / 三废排放 / 自然资源依赖
  - 社会（S）：劳工纠纷 / 供应链人权 / 数据隐私 / 产品安全
  - 治理（G）：董事会独立性 / 关联交易 / 信息披露质量
- 是否有重大环境 / 劳工 / 治理事件历史？
- 欧盟 CSRD（企业可持续发展报告指令）/ SEC 气候披露规则 / 中国 ESG 信披的影响？
  - 合规成本估算？
  - 数据披露能否过审？
- 主要 ESG 评级机构（MSCI / Sustainalytics / FTSE）的当前评级及趋势？
- ESG 评级被下调可能触发哪些被动资金赎回？

### 3.8 地缘 / PCAOB / 退市风险（v2 新增）
- 中美关系恶化时的影响？（关税 / 出口管制 / 投资限制 / 实体清单）
- 是否在 PCAOB 风险名单上？（即 modifier_flags 含 `pcaob_risk`）
- 主要市场被监管行动的概率？
  - 美国：HFCAA 退市风险 / SEC 调查 / 司法部 FCPA 调查
  - 中国：网信办 / 反垄断 / 数据出境
  - 欧盟：GDPR / DMA / CSRD

- 若被强制退市 / 限制 ADR 交易，回流港股 / A 股的可行性？

## 九、预先尸检与 scorecard.json 联动（强制三字段同步）

本文件的预先尸检结果必须反映到 scorecard.json 的以下**三个字段**，agent 必须在 stage 4 自检阶段交叉校验：

### 9.1 联动字段 1：`veto_check.value_trap_signals_triggered`

- 规则：若 §一 概率累计 > 40%（即 5 年内亏 50% 的累计概率 > 40%），则 `value_trap_signals_triggered` 应给予 **≥ 4**
- 若 §五 致命单点风险命中 ≥ 2 条，`value_trap_signals_triggered` 应给予 **≥ 5**
- 一旦 `value_trap_signals_triggered ≥ 5`，V2_SPEC §五 一票否决触发 → 强制 PASS

### 9.2 联动字段 2：`scorecard.risk.score`

- 规则：若 §五 致命单点风险命中任意 1 条 → `risk.score` 至少扣 2 分

### 9.3 联动字段 3：`decision.rationale`
