# 风险评估与反偏差参考（v2 - 强制门禁）

## 一、设计原则（v2 强化）

- 价值投资 50% 的回报来自避雷
- 本模块在 skill 中作为**前置强制门禁**——未完成不输出投资结论
- v2 关键升级：
  - 能力圈判定改为"硬条件 + 软计数 + 行业适配"
  - 反向证据从"字数配额"改为"三件套"
  - 行为偏差从 5 项扩展到 8 项
  - 风险分层加入 ESG / AI 替代 等 v2 新增维度

### 1.1 v2 改动的实战触发原因（必读）

| v1 问题 | 实战案例 | v2 修复 |
|---|---|---|
| 能力圈"任一触发即 too-hard"过严 | 百济神州触发 5/6 项，被错判为 too-hard；A 股 70% + 全球 90% 前沿科技公司全部归 too-hard | §五 改为"硬+软计数 + 行业适配" |
| 反向证据"字数配额"被 AI 用"虚拟代表 X 风格"凭空捏造 | 百济 v1 报告 §6.4 第 1/3/4 条均标注"虚拟代表 Spruce Point 风格"；神华、BRK.B 类似 | §六 改为"具名 + 数字 + 概率重定标"三件套 |
| 5 大行为偏差不够用 | 10 份样本中真正咬住决策的是群体思维 / 叙事偏差 / 可得性偏差 | §八 扩展到 8 项 |
| 风险分层未覆盖新型威胁 | AI 替代 / ESG / 数据泄露 v1 缺位 | §二 新增第 5 层"新型风险" |

### 1.2 价值投资三大避雷信条

> "我们不是要做出绝顶聪明的决策，而是要避免犯极其愚蠢的错误。" —— Charlie Munger
>
> "First, do no harm." —— 医学希波克拉底誓言（巴菲特多次引用于风控语境）
>
> "Rule No.1: Never lose money. Rule No.2: Never forget rule No.1." —— Warren Buffett

## 二、风险分层框架（v2 扩展 5 层）

| 层级 | 含义 | 价值投资视角的警惕信号 |
|---|---|---|
| **公司特有** | 个股风险 | 应收账款增速 > 营收 / 经营现金流持续低于净利润 / 关联交易 > 10% / 频繁更换审计师/CFO / 股权质押率 > 50% / 商誉/净资产 > 30% |
| **行业** | 行业系统性 | 产能利用率持续下行 / CR5 快速变化 / 替代技术 / 上下游议价权转移 / 整体 ROIC 跌破 WACC |
| **宏观/政策** | 国家级 | 列入反垄断 / 数据安全 / 双碳 / 集采 重点名单；利率周期；行业税收优惠到期 |
| **地缘/汇率** | 跨国级 | 单一国家收入 > 30% / 关键供应链海外集中 / 外币负债敞口 / 出口被列入制裁 |
| **新型风险**（v2 新增） | AI / ESG / 网络安全 / 气候诉讼 | AI 替代核心业务 / ESG 评级下调 / 重大数据泄露 / 气候相关诉讼 / 供应链单点失败 |

### 2.1 每层最少自查问题数

- **公司特有**：≥ 6 项核查
- **行业**：≥ 4 项核查
- **宏观/政策**：≥ 3 项核查（A 股标的强制 ≥ 5 项）
- **地缘/汇率**：≥ 3 项核查（含跨境业务的强制 ≥ 5 项）
- **新型风险（v2）**：≥ 3 项核查

### 2.2 输出位置

5 层风险评估必须出现在 `full-report.md §6` 中，并按层分小节展示，禁止合并为一段。

### 2.3 各层核查清单（强制全跑）

#### 公司特有（6 项最少）

1. 应收账款 / 营收比值 5 年趋势（同比、环比）
2. 经营现金流 / 净利润 比值 3 年平均
3. 关联交易披露完整性（占营收、占采购比例）
4. 审计机构 / CFO 变更频率（10 年内 ≥ 2 次为警示）
5. 股权质押率（大股东、董事会成员）
6. 商誉 / 净资产 比值
7. 表外负债（商票、对赌、SPV、担保）
8. 衍生品 / 套保净敞口（金融类、商品类）

#### 行业（4 项最少）

1. 产能利用率 / 库存周转 / 应收账款天数
2. CR5 / CR10 集中度变化
3. 替代技术成熟度（5 年技术路线图）
4. 上下游议价权（毛利率 vs 议价能力）
5. 整体行业 ROIC vs WACC

#### 宏观/政策（A 股 5 项最少）

1. 是否列入反垄断 / 数据安全 / 双碳 / 集采名单
2. 行业税收优惠到期年份
3. 当前利率周期位置
4. 国家产业政策方向（5 年规划、十四五 / 十五五）
5. 行业是否处于政策"修正期"（如教培前夜、互联网平台前夜）

#### 地缘/汇率（跨境 5 项最少）

1. 收入地区分布（单一国家 > 30% 警示）
2. 关键供应链海外集中度
3. 外币负债敞口 / 自然对冲
4. 是否被列入制裁 / 实体清单 / UVL
5. VIE 结构稳定性（中概股）

#### 新型风险（v2 必跑 3 项）

1. AI 替代风险评分（核心岗位被 AI 替代概率）
2. ESG 评级（MSCI / Sustainalytics / 中证 ESG）
3. 重大数据 / 网络安全事件历史

## 三、10 + 5 大黑天鹅清单（v2 扩展）

### v1 保留 10 项

1. **监管反转**（教培式归零）—— 例：2021 年 "双减"使新东方等 90% 一夜消失
2. **技术颠覆**（柯达 / 诺基亚式）—— 例：数码相机对柯达、iPhone 对诺基亚
3. **关键人物风险**（创始人 / 核心技术人员）—— 例：苹果之于乔布斯、特斯拉之于马斯克、伯克希尔之于巴芒
4. **单一大客户依赖**（> 20% 警报，> 40% 直接 PASS）—— 例：歌尔之于苹果耳机订单波动
5. **表外负债**—— 例：恒大商票 / VIE 担保 / SPV 抽屉协议
6. **商誉减值地雷**—— 商誉 / 净资产 > 30% 列为预警
7. **对赌 / 业绩承诺到期**—— 并购 3 年承诺期最后一年常爆雷
8. **可转债 / 定增稀释**—— 摊薄 EPS 与 ROE
9. **跨境法律风险**（VIE / PCAOB / 退市）—— 例：滴滴 ADR 退市、阿里 VIE 不确定性
10. **能源 / 原材料断供**—— 例：俄乌战争后欧洲能源、稀土被禁运

### v2 新增 5 项

11. **AI 替代** — AI 是否能取代公司核心岗位/业务（如：律所、客服、设计、初级程序员、翻译、内容运营）
    - 触发警示：核心业务 50% 以上是"可被 GPT-5 / Claude Opus 完成"的标准化白领劳动
    - 案例：Chegg 因 ChatGPT 一年股价腰斩；Stack Overflow 流量崩盘
12. **ESG 黑天鹅** — 重大环境事故 / 强迫劳动指控 / 监管处罚
    - 欧盟 CSRD（公司可持续报告指令 2024 年生效）
    - SEC 气候披露规则（2024-03 通过）
    - 美国 UFLPA（维吾尔强迫劳动预防法）
    - 案例：BP 漏油事件市值蒸发 50%
13. **重大数据泄露** — GDPR / CCPA / 国内《个人信息保护法》高额罚款 + 用户流失
    - GDPR 罚款上限 4% 全球营收（Meta 已被罚 12 亿欧元）
    - 案例：Equifax 2017 泄露 1.47 亿用户后赔偿 7 亿美元
14. **气候相关诉讼** — 强制 ESG 披露后衍生的股东集体诉讼
    - 趋势：美国证券集体诉讼 2024 年 30% 涉及 ESG 虚假陈述
    - 案例：荷兰 Shell 被法院判决加速减碳
15. **供应链单点失败** — 半导体 / 关键化学品 / 罕见金属 单一来源
    - 案例：台积电之于全球 AI 芯片、ASML 之于 EUV 光刻机、金川镍 / 中国稀土
    - 触发警示：单一供应商 / 单一地区 > 70%

### 3.1 输出要求

每项黑天鹅需在 `full-report.md §6` 给出"是 / 否 / 不适用 + 一句话证据"，禁止整体合并标记"已检查"。

### 3.2 黑天鹅过审表模板

| # | 黑天鹅 | 是否触发 | 一句话证据 / 来源 | 概率估计 |
|---|---|---|---|---|
| 1 | 监管反转 | 否 | 当前行业未列入重点整治名单 | < 5% |
| 2 | 技术颠覆 | 警示 | AI Agent 可能替代部分客服 / 翻译岗位 | 15-25% |
| 3 | 关键人物 | 否 | CEO 已有明确接班人计划 | < 5% |
| ... | ... | ... | ... | ... |

### 3.3 触发等级定义

- **是 / 强触发**：5 年内主观概率 > 30%，需在预先尸检中作为主要失败路径
- **警示 / 中触发**：5 年内主观概率 10-30%，需在敏感度分析中体现
- **否 / 弱触发**：5 年内主观概率 < 10%，可标记备查
- **不适用**：行业 / 商业模式上根本不存在该风险（需注明理由）

## 五、能力圈判定（v2 升级为"硬+软计数法"）

### 5.0 v1 问题回顾

v1 的"任一触发即 too-hard"将百济神州（前沿生物制药）、Temu（监管多变）、宁德时代（技术路线快速演进）等普遍归 too-hard——而这些恰恰是 A 股 / 中概股投资者最想分析的对象。**百济触发 5/6 项即被归类**，属过严失效。

v2 改为 **硬条件 + 软计数 + 行业适配** 三层判定。

### 5.1 硬条件（任一触发 → 强制 too-hard）

- **#1**：无法用 3 句话向高中生解释商业模式
  - 通过标准："公司卖什么 / 卖给谁 / 怎么赚钱"三问全部用一句话回答清楚
  - 失败示例：合成生物学 + 多业态 + 跨境 SaaS 的高度复合公司
- **#2**：核心技术超出 AI 自身训练数据范围
  - 例：超前沿量子物理初创、未公开军工技术、依赖 < 1 年的最前沿论文且未进入主流综述
  - 通过标准：用 ToolSearch 检索 3 篇以上中英文综述，能复述技术路径主线

### 5.2 软条件（计数法，≥ 2 项触发 → too-hard）

- **#3**：行业 5 年内技术路线发生过 ≥ 2 次颠覆
- **#4**：收入确认规则需要专业会计师解释（如：CDMO / SaaS 多年合同 / 控股集团 look-through）
- **#5**：估值高度依赖单一未上市变量（如：下一代产品获批、并购完成、单一诉讼判决）
- **#6**：监管框架在过去 24 个月发生重大变化（且方向不清晰）

### 5.3 行业适配条款（v2 关键）

不同行业的"软条件"权重不同，按以下规则修正：

| 行业标签 | 修正规则 | 理由 |
|---|---|---|
| `biotech_unprofitable` | **不计算 #3 和 #5** | 前沿性 / 颠覆性是行业本质，管线获批是商业模式核心，非"问题" |
| `high_growth_platform`（如 Temu 国际化、SHEIN） | **不计算 #6** | 跨境监管变化是行业常态，公司有应对预案即可 |
| `A 股政策敏感行业`（教育 / 医药 / 互联网 / 地产 / 新能源） | **#6 改读为"政策方向是否清晰"** | 清晰但严格（如双碳）≠ too-hard；方向不清（如教培前夜）= too-hard |
| `holding_conglomerate`（控股集团，如 BRK.B / 长江实业） | **默认满足 #1，但需重点评估 #4** | 商业模式可解释但 look-through earnings 复杂 |
| `cyclical_commodity`（煤炭 / 钢铁 / 油气 / 航运） | **默认满足全部硬条件，主要看周期判断而非能力圈** | 商业模式简单，难点在周期位置 |
| `fintech_with_regulatory_overhang`（蚂蚁 / 京东金融） | **#6 必须强读，不豁免** | 监管是核心变量 |
| `frontier_hardware`（量子计算 / 商用核聚变 / 脑机接口） | **#2 必须强读** | 大概率触发硬条件 #2 → too-hard |

### 5.4 判定示例（覆盖 10 份样本中至少 5 家）

| 公司 | 行业标签 | 硬条件触发 | 软条件触发 | 判定结果（v2） | 理由 |
|---|---|---|---|---|---|
| **百济神州** | biotech_unprofitable | 0 | #4（仅 1 项，#3/#5 被豁免） | **不 too-hard** | biotech 豁免后仅 1 项软条件，可继续分析 |
| **万科 A** | A 股政策敏感（地产） | 0 | #4（应收 vs 合同负债复杂）+ #6（地产政策方向不清） | **暂时 too-hard** | 软条件 ≥ 2 触发，待行业稳定后重评 |
| **贵州茅台** | 消费品 | 0 | 0 | **不 too-hard** | 商业模式极简，软硬均不触发 |
| **伯克希尔 BRK.B** | holding_conglomerate | 0 | #4（look-through 略复杂，可解） | **不 too-hard** | 仅 1 项软条件触发 |
| **拼多多（含 Temu）** | high_growth_platform | 0 | #5（Temu 国际化未上市变量）—— #6 被豁免 | **不 too-hard** | 仅 1 项软条件，可继续分析 |
| **宁德时代** | 新能源（技术演进快） | 0 | #3（钠电 / 固态电池 / LFP 路线切换） | **不 too-hard** | 仅 1 项软条件，可继续分析 |
| **中国神华** | cyclical_commodity | 0 | 0（行业适配豁免） | **不 too-hard，但需周期判断** | 适配规则覆盖 |
| **某商用核聚变初创** | frontier_hardware | #2 触发 | — | **强制 too-hard** | 硬条件 #2 直接归类 |

### 5.5 判定流程图（伪代码）

```
def capability_circle_v2(company, industry_tag):
    # Step 1: 硬条件
    if hard_1_business_model_unclear(company):
        return "too-hard (硬 #1)"
    if hard_2_beyond_ai_training_data(company):
        return "too-hard (硬 #2)"

    # Step 2: 行业适配过滤
    soft_conditions = [3, 4, 5, 6]
    if industry_tag == "biotech_unprofitable":
        soft_conditions = [4, 6]
    elif industry_tag == "high_growth_platform":
        soft_conditions = [3, 4, 5]
    elif industry_tag == "cyclical_commodity":
        return "not_too_hard_but_cycle_dependent"
    # ... 其他规则

    # Step 3: 软条件计数
    triggered = count_soft_triggered(company, soft_conditions)
    if triggered >= 2:
        return f"too-hard (软条件 {triggered} 项)"
    return f"not_too_hard (软条件仅 {triggered} 项)"
```

### 5.6 输出位置

能力圈判定结论必须出现在 `full-report.md §6.1`，并明确给出：

- 行业标签（来自 5.3 列表）
- 硬条件检查结果（2 项）
- 软条件检查结果（4 项及行业适配后剩余项数）
- 最终判定（too-hard / not-too-hard）

## 六、反向证据搜索（v2 改为"三件套"，废除字数配额）

### 6.1 v1 字数配额已被 AI 凭空捏造凑数

实战发现：

- **百济神州 v1 报告 §6.4** 第 1/3/4 条均标注 **"虚拟代表 Spruce Point 风格"**——空头报告凭空虚构，无任何真实出处
- **中国神华 v1 报告 §6.4** 第 2 条用"行业普遍悲观情绪"凑字数
- **BRK.B v1 报告 §6.4** 用"假设 Hindenburg 会这样写"——同样捏造

字数配额的副作用：AI 为完成"30% 篇幅"指标，**宁可虚构也要凑够**。这反而违背了反向证据的本意（寻找真实异议）。

### 6.2 v2 三件套规则（强制）

每条反向证据必须满足以下**三件套**，缺一不可：

#### 件 1：具名出处

- 明确的机构 / 个人名字 + 报告标题 + 时间
- 正确示例：
  - "Hindenburg Research 2024-08-15 报告《XYZ Capital - The Master Of Manipulation》"
  - "GMT Research 2023-11 报告《China Vanke: Accounting Red Flags》"
  - "雪球用户 @灰熊投研 2024-03-22 帖文（获赞 287）"
  - "前 CFO Jane Doe 在 2024-05-10 Financial Times 访谈"
- **错误示例（严禁出现）**：
  - "虚拟代表 Spruce Point Capital 风格"
  - "假设 Hindenburg 会这样写"
  - "做空机构观点"
  - "行业普遍悲观"
  - "市场上有人质疑"

#### 件 2：具体数字预测

- 必须含至少一条数字预测或量化判断
- 正确示例：
  - "Hindenburg 预测 ROE 3 年内从 22% 降至 8%"
  - "GMT 估算调整后 PB 应 < 0.5×，对应股价下行 60%"
  - "做空报告测算应收账款虚增 ¥40 亿，占总资产 12%"
  - "离职 CFO 暗示真实毛利率 < 公布值 15 个百分点"
- **错误示例（严禁出现）**：
  - "普遍看空"
  - "行业前景悲观"
  - "估值偏高"（无具体倍数）
  - "增长不可持续"（无具体指标）

#### 件 3：概率重定标

- 基于该看空观点，重新估计"投资亏损 50% 的主观概率"
- 正确示例：
  - "按 Hindenburg 看空逻辑，亏损 50% 的主观概率应从 15% 上调至 28%"
  - "若 GMT 关于商誉减值的判断为真，3 年内股价腰斩概率 ≥ 35%"
- **错误示例（严禁出现）**：
  - "风险加大"（无具体概率）
  - "需谨慎"

## 七、能力圈与卖出纪律（保留 v1 + v2 强化）

### 7.1 合理卖出理由（按优先级）

1. **投资逻辑被证伪**（商业模式 / 护城河出问题）—— v2 必须给具体"被证伪"指标
   - 例：ROE 连续两年跌破 8%
   - 例：CR5 市占率 3 年下降 5pp 以上
   - 例：核心产品市场份额跌破历史中位数
2. **管理层重大失信**（造假 / 关联输送 / 能力暴露）
3. **严重高估**（DCF 隐含价值的 1.5-2x 以上）
4. **发现更好机会**（机会成本，要求新机会回报率高出当前 ≥ 30%）

### 7.2 禁止的卖出理由（v2 强化）

- 涨了 / 跌了多少（情绪驱动）
- 持有多久（时间锚定）
- 短期波动恐慌
- "周围人都卖了"（群体思维，见 §八）
- "财经媒体说要崩了"（可得性偏差）

### 7.3 输出要求

卖出触发条件清单需明确写入 `full-report.md §0.5 与 §11.4`，并在 `templates/decision-summary.md` 中固定为可勾选清单。

## 八、行为偏差对照表（v2 扩展为 8 项）

## 九、风险评分与一票否决（保留 v1，v2 加显式标注规则）

### 9.1 评分门槛

- **风险维度评分 ≤ 3** → 强制 PASS（V2_SPEC §五.3）
- **风险维度评分 ≤ 2** → 不仅 PASS，且建议在主报告 §0.5 与 §11.4 显式标注 **"重大致命单点 - 请勿持有"**
- **能力圈 too-hard 触发** → 强制 PASS
- **价值陷阱信号 ≥ 5 项触发** → 强制 PASS

### 9.2 一票否决清单（任一即 PASS）

- 单一客户 > 40%
- 应收账款 > 营收 12 个月
- 商誉 / 净资产 > 50%
- 股权质押率 > 70%
- 审计机构 12 个月内更换 ≥ 2 次
- CFO 24 个月内更换 ≥ 2 次
- 经营现金流 / 净利润 比值 3 年平均 < 0.5
- 列入 SEC / 证监会重大调查（未结案）
- 被具名做空且 ≥ 1 条三件套未被公司有效反驳

### 9.3 评分输出格式

```
风险维度评分：X / 10

- 公司特有风险：X / 10（含 N 项核查）
- 行业风险：X / 10
- 宏观/政策风险：X / 10
- 地缘/汇率风险：X / 10
- 新型风险（v2）：X / 10

一票否决项触发：[列表 / 无]
最终判定：BUY / HOLD / PASS / PASS-重大致命单点
```
