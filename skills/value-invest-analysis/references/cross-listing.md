# 跨市场上市决策框架（v2 新增）

> **作用域**：本文件是 AH 双重上市决策、VIE 折价分级、双重上市分类、PCAOB 风险评估、跨市场股息税计算的**唯一真理源**。`market-cn.md` / `market-hk.md` / `market-us.md` / `valuation.md` / `red-flags-checklist.md` 在涉及上述主题时必须指向本文件，禁止再发明分类或独立计算。

---

## 一、为什么需要本文件

### 1.1 v1 评审发现的核心痛点

- **AH 决策混乱**：`market-hk.md` AH 章节仅 5 行导致 5 份 AH 样本中 4 份处理混乱（工行 / 招行 / 比亚迪 / 神华 / 万科 在 H/A 选择上各执一词）
- **VIE 折价不统一**：v1 样本中腾讯给 5-10% / PDD 给 25% / 百济给 0%，缺乏统一分级框架
- **上市分类基础错误**：v1 样本将腾讯标为"双重主要上市"（错误，腾讯仅港股主要）、百济标为"二次上市"（错误，实为美港双重主要）
- **股息税计算遗漏**：港股通持 H 股的"自然人 20% 内地税"在 v1 多次遗漏，导致估算误差 ~10pct
- **PCAOB 风险无定价**：v1 对中概 ADR 退市风险既没等级又没估值调整

### 1.2 本文件的统一处理目标

- **AH 决策**：6 维评分通过后，统一决策树决定买 A 还是 H
- **VIE 分级**：4 类公司 → 固定折价区间，禁止个案发明
- **上市分类**：4 个互斥分类，每个公司必归其一
- **股息税**：完整对照表覆盖中国个人 / 港股通 / 海外 / ADR 所有持有路径
- **PCAOB 风险**：4 个等级 → 估值调整 + `required_pct` 加成

## 二、上市结构四分类（v2 锁定，禁止再发明分类）

任何中概或 AH 公司必须被归入下表**唯一一类**。这是 v2 跨市场估值的入口判别。

| 分类（机器标签） | 定义 | 典型案例 | 估值含义 |
|---|---|---|---|
| `dual_primary_listing` 双重主要上市 | 两个交易所都是"主要上市地"，财报独立提交两地监管 | 阿里 9988.HK + BABA / 百济 6160.HK + ONC / 京东 9618.HK + JD / 网易 9999.HK + NTES / 哔哩哔哩 9626.HK + BILI | 流动性最佳，相同公司，两地估值会趋同，**PCAOB 风险部分隔离** |
| `secondary_listing` 二次上市 | 仅在原市场是主要上市，新市场是补充（多采用 SPAC / SPV 简化挂牌） | 部分早期回港中概股 | 流动性较差，估值仍跟随主市场，**PCAOB 风险全暴露** |
| `adr_listed` ADR 美国存托凭证 | 仅在美国通过 ADR 上市，无对应主要上市 | 腾讯 TCEHY（OTC，非主要）/ 阿里早期 BABA 单独 | 受 PCAOB 影响大，可能折价 5-25%；OTC ADR 流动性次之 |
| `a_h_dual_listed` A+H 双重上市 | A 股 + H 股同时挂牌（**不是**双重主要上市，是中国境内同股双地） | 工行 / 招行 / 比亚迪 / 中海油 / 神华 / 万科 / 中石油 / 中石化 / 平安 / 农行 / 建行 / 中行 / 交行 等 | 长期溢价（A 股）vs 折价（H 股），按 AH 决策树选择 |
| `single_a_listing` 仅 A 股 | 无 H / 无 ADR | 茅台 600519 / 五粮液 / 海康 / 宁德 等 | 无跨市场选择，直接定价 |
| `single_h_listing` 仅港股 | 无 A / 无 ADR | 腾讯 0700（主要上市港）/ 美团 3690 / 小米 1810 / 友邦 1299 | 无跨市场选择，按港股定价 |

> **重要纠错**：
> - **腾讯**：港股 0700.HK 为唯一主要上市地，TCEHY 为美国 OTC ADR（非主要上市）→ 分类为 `single_h_listing`（主类）+ 同时有 `adr_listed` 补充
> - **百济神州**：港股 6160.HK + 美股 ONC 均为主要上市 → `dual_primary_listing`（v1 误标二次）
> - **阿里巴巴**：港股 9988.HK + 美股 BABA 均为主要上市（2022 年完成双重主要上市转换）→ `dual_primary_listing`

## 三、AH 双重上市决策树（v2 核心）

### 3.1 步骤 1：查询当前 AH 溢价

- **数据源**：恒生 AH 溢价指数（Hang Seng Stock Connect China AH Premium Index，代码 HSAHP）
- **长期中枢**：130-150（即 A 股相对 H 股长期溢价 30-50%）
- **5 年区间**：通常 120-155
- **极值**：2024 年初一度 > 155，2018 年最低 ~115

> **使用方式**：取当前 AH 溢价指数 → 计算 5 年分位（百分位）→ 输入决策树步骤 3。

### 3.2 步骤 2：股息税完整对照表

下表是跨市场股息计算的**唯一基准**，禁止在 market-cn / market-hk / market-us 中重复定义。

| 持有方 → 资产类型 | A 股 | H 股（直接持有，香港账户/海外账户） | 红筹股 H | 港股通持 H 股 | 美国 ADR |
|---|---|---|---|---|---|
| 中国居民个人 持有 < 1 月 | **20%** | 不适用 | 不适用 | 不适用 | 不适用 |
| 中国居民个人 1 月-1 年 | **10%** | 不适用 | 不适用 | 不适用 | 不适用 |
| 中国居民个人 持有 > 1 年 | **0%** | 不适用 | 不适用 | 不适用 | 不适用 |
| 中国居民个人 通过港股通持 H | — | — | — | **港 10% + 内地 20% = 30%**（自然人差额征收） | — |
| 中国境内机构（公募 / QDII） | 0%（持股 > 1 年） | 10% | 0% | 10%（机构差额暂免） | 30% 预提（条约可降）|
| 香港居民 / 海外个人持有 H | — | **10%**（H 股内地税）| **0%** | — | — |
| 美国居民个人持有 ADR | — | — | — | — | **30% 预提**（中美无 100% 减免税收协定，部分可至 10%）|

**关键计算结论**：

1. **A 股长期持有股息税 0%** vs **H 股 10%** → A 股股息净收益的"税后乘数"比 H 股高 ~10%
2. 但 A 股价格通常比 H 股高 30-50%（AH 溢价指数 130-150）→ **A 股股息率（dividend yield）显著低于 H 股**
3. **港股通持 H 股**：股息率高（接近 H 股直接持有），但自然人 + 20% 内地差额税 = **总 30% 股息税**，劣于直接香港账户持 H 股（10%）
4. **红筹股（公司注册在开曼 / 百慕大）**：H 股 10% 内地税不适用，**香港居民 / 海外个人 0%**；但港股通持有时仍按 20% 自然人差额税

### 3.3 步骤 3：AH 决策树（详细）

```
输入：
  d_A   - A 股年股息率（最近一期股息 / A 股价）
  d_H   - H 股年股息率（最近一期股息 / H 股价）
  p     - 当前 AH 溢价（A/H - 1，例如 0.40 表示 A 股贵 40%）
  p_pct - 当前 AH 溢价在 5 年的百分位（0-100）
  holder_type - 持有人税务身份
  liquidity_h - H 股流动性评分（0-1，日均成交额）
  flags - 风险标记（如 distressed / pcaob_risk / theme_premium）
  v2_score - 6 维评分（已通过）

═══════════════════════════════════════════
step 1: 不分红 / 低分红公司（d_A < 2% 且 d_H < 2%）
───────────────────────────────────────────
  → AH 决策几乎不影响最终回报
  → 按流动性 + 资金面选：
    - A 股流动性通常更好（散户活跃）
    - 但港股通"双柜台"H 股有南向资金长期净流入
    - 港股通额度风险（限购）少见
  → **默认推荐 A 股**（流动性 + 估值上行空间）
  → 例外：A 股有显著"题材 / 次新 / 消息"溢价 → 转 H 股

═══════════════════════════════════════════
step 2: 高分红公司（d_A ≥ 2% 或 d_H ≥ 2%）
───────────────────────────────────────────
  计算"长期持有后年化股息净回报":

  # A 股端
  if holder_type == 'mainland_long':       # 中国居民 > 1 年
      return_a = d_A * 1.00
  elif holder_type == 'mainland_medium':   # 1 月-1 年
      return_a = d_A * 0.90
  elif holder_type == 'mainland_short':    # < 1 月
      return_a = d_A * 0.80
  else:                                    # 海外通过 QFII / 沪深通
      return_a = d_A * 0.90                # QFII 10%

  # H 股端
  if holder_type == 'stock_connect':       # 大陆个人通过港股通
      return_h = d_H * 0.70                # 10% HK + 20% 内地差额
  elif holder_type in ('oversea_personal', 'oversea_institution'):
      return_h = d_H * 0.90                # 海外个人/机构持 H 10%
      if 是红筹股:
          return_h = d_H * 1.00            # 海外持红筹股免税
  else:
      return_h = d_H * 0.90

  决策核心：
  if return_h - return_a > 0.005:          # H 股至少高 0.5pct 才值切换
      推荐 H 股（前提：流动性可接受）
  elif return_a - return_h > 0.005:
      推荐 A 股
  else:
      → 进入 step 3 看 AH 溢价分位

═══════════════════════════════════════════
step 3: AH 溢价分位调整
───────────────────────────────────────────
  if p_pct > 80:        # 5 年高位
      → H 股有均值回归机会，加权 H 股（+10% 权重）
  elif p_pct < 20:      # 5 年低位
      → AH 价差已偏小，A 股流动性优势凸显，加权 A 股（+10% 权重）
  else:
      → 维持 step 1 / step 2 结果

═══════════════════════════════════════════
step 4: 风险加成
───────────────────────────────────────────
  if liquidity_h < 0.3:                    # H 股日均成交 < 1 亿港元
      → 强制选 A 股（无论前面结果）
  if 'theme_premium' in flags:             # A 股"消息驱动"溢价
      → 强制选 H 股（A 股可靠性低）
  if 'distressed' in flags:                # 已现金流危机
      → 两个都不推荐，列入观察名单
  if 'pcaob_risk' in flags and holder_type 含美:
      → A 股不受影响，加权 A 股

═══════════════════════════════════════════
输出：'A股' | 'H股' | '弃' + 决策理由
═══════════════════════════════════════════
```

### 3.4 决策树代码化（伪代码，机器可执行）

```python
def ah_choice(
    d_a: float,
    d_h: float,
    premium_pct: float,           # 0.40 表示 A 比 H 贵 40%
    premium_percentile: float,    # 0-100，5 年百分位
    holder_type: str,             # 'mainland_long' | 'mainland_medium' | 'mainland_short'
                                  #   | 'oversea_personal' | 'oversea_institution'
                                  #   | 'stock_connect' | 'qdii'
    liquidity_score_h: float,     # 0-1
    flags: set,                   # {'theme_premium', 'distressed', 'pcaob_risk', ...}
    is_red_chip: bool = False,
) -> tuple[str, str]:
    """返回 (选择, 理由)"""

    # ─── 风险硬否决（优先级最高）───
    if 'distressed' in flags:
        return ('弃', 'distressed flag，观察名单')

    # ─── step 1: 不分红 / 低分红 ───
    if d_a < 0.02 and d_h < 0.02:
        if 'theme_premium' in flags:
            return ('H股', 'A 股有题材溢价，H 股更可靠')
        if liquidity_score_h < 0.3:
            return ('A股', 'H 流动性差且都不分红')
        return ('A股', '不分红 → 流动性优先')

    # ─── step 2: 计算税后股息回报 ───
    a_tax_mult = {
        'mainland_long': 1.00,
        'mainland_medium': 0.90,
        'mainland_short': 0.80,
        'oversea_personal': 0.90,
        'oversea_institution': 0.90,
        'qdii': 0.90,
        'stock_connect': 1.00,   # 港股通买 A 不存在；默认按内地长持
    }.get(holder_type, 0.90)

    if holder_type == 'stock_connect':
        h_tax_mult = 0.70
    elif holder_type in ('oversea_personal', 'oversea_institution'):
        h_tax_mult = 1.00 if is_red_chip else 0.90
    else:
        h_tax_mult = 0.90

    return_a = d_a * a_tax_mult
    return_h = d_h * h_tax_mult

    # ─── step 4: 流动性硬否决（提前到这里）───
    if liquidity_score_h < 0.3:
        return ('A股', f'H 流动性差（score={liquidity_score_h:.2f}）')

    # ─── 决策核心 ───
    delta = return_h - return_a

    # ─── step 3: AH 溢价分位调整（折算成等效 delta）───
    if premium_percentile > 80:
        delta += 0.003       # 加权 H 股 30bp
    elif premium_percentile < 20:
        delta -= 0.003       # 加权 A 股 30bp

    # ─── 风险加成 ───
    if 'theme_premium' in flags:
        delta += 0.005       # 强加权 H
    if 'pcaob_risk' in flags and holder_type.startswith('oversea'):
        delta -= 0.005       # 海外持有人 pcaob 时 H 也受影响（如果 H 是中概） → 这里偏 A
                             # 注：a_h_dual_listed 不受 pcaob 影响，本调整一般不触发

    if delta > 0.005:
        return ('H股', f'税后股息回报 H-A = {delta*100:.1f}pct')
    elif delta < -0.005:
        return ('A股', f'税后股息回报 A-H = {-delta*100:.1f}pct')
    else:
        # 接近持平 → 默认 A 股（流动性 + 国内资金面）
        return ('A股', '股息回报接近，A 流动性优先')
```

## 四、VIE 折价分级表（v2 核心）

### 4.1 VIE 是什么

VIE（Variable Interest Entity，可变利益实体）是中国互联网 / 教育等限制外资行业为绕开外资限制设计的法律架构：
- 注册在开曼 / 百慕大的"上市公司"（外资可控）
- 通过协议控制（VIE 协议）合并国内运营实体
- 国内监管定性"灰色合法"，2021 年教培双减后协议风险骤升

### 4.2 四级分级表（v2 锁定，禁止个案发明）

| 公司分类 | 触发条件 | VIE 折价建议 | 案例 | 备注 |
|---|---|---|---|---|
| **核心 VIE** | 主营收入 > 70% 来自 ICP 牌照、教育、新闻、电信、文化等强外资限制行业 | **20-25%** | BABA / 9988（电商 ICP） / BIDU / 9888（搜索 ICP） / EDU / 9901（教育，已转型） / YY / DIDI（出行 + 数据安全）| 风险包括：监管定性、合规暂停、ADR 退市 |
| **部分 VIE** | 个别业务用 VIE（< 50% 收入），主体在外资可控行业 | **10-15%** | TCEHY / 0700（游戏、社交部分 VIE，其余非 VIE 持牌运营）/ PDD（电商主体 + 物流/支付）/ 9618 京东（部分牌照 VIE） | 风险局部，可解耦 |
| **非 VIE 中概** | 直接控股，无协议控制，行业不在外资限制清单 | **5-10%** | 6160 百济（生物医药）/ ZTO 中通（快递牌照持有）/ TIGR 老虎（券商持牌）/ NIO 蔚来（部分）| 仅承担"中概标签"的市场情绪折价 |
| **非中概外资股** | 全球普通公司，无中国架构风险 | **0%** | 苹果 / 微软 / 友邦 1299（百慕大注册但港股主要上市）| 不适用本表 |

### 4.4 折价应用方式

```
fair_value_after_vie = fair_value_pre_vie × (1 - vie_discount)
```

VIE 折价**仅应用于估值终值**，不应用于 EV/EBITDA 倍数本身（避免双重折价）。

## 五、双重主要上市 vs 二次上市判别表

### 5.1 判别表

| 特征 | 双重主要上市 `dual_primary_listing` | 二次上市 `secondary_listing` |
|---|---|---|
| 财报独立提交（两地监管） | ✓ | ✗（仅主市场提交完整财报）|
| 港股通可纳入 | 大概率 ✓（满足市值流通条件即可）| 不必然（需符合特殊纳入条件）|
| 流动性 | 两地都高 | 主市场高，另一地低 |
| PCAOB 风险（中概） | 部分隔离（港股不受 PCAOB 直接管辖）| 全暴露（仍主要受美国监管影响）|
| 估值趋同性 | 强（套利空间小） | 弱（主市场拉动）|
| 退市风险传导 | 仅美股 ADR 退市，港股仍运作 | 主市场退市 → 二次上市地或受波及 |
| 内地国资 / 公募可买入 | 港股通入选后可买 | 港股通入选门槛较高 |

### 5.2 典型案例

**dual_primary_listing**：阿里巴巴 9988+BABA、京东 9618+JD、百济神州 6160+ONC、网易 9999+NTES、哔哩哔哩 9626+BILI、知乎 2390+ZH、小鹏 9868+XPEV、理想 2015+LI、蔚来 9866+NIO

**secondary_listing**：早期回港中概的部分公司（多数已转换为双重主要）

### 5.3 v1 评审纠错

## 六、PCAOB 风险评估（v2 引入）

### 6.1 背景

PCAOB（美国公众公司会计监督委员会）2021 年起对中国会计师事务所的检查权产生争议，HFCAA 法案要求三年内不能审计即强制退市。2022 年底中美签署协议改善，但风险并未消除：
- 任一年度抽查失败仍可触发退市倒计时
- 行业敏感公司（教育 / 数据 / AI）仍是高风险靶子

### 6.2 四级风险分级表

| 公司分类 | PCAOB 风险等级 | 估值调整 | `required_pct` 加成 | 代表 |
|---|---|---|---|---|
| **单一在美 ADR**（无双重 / 二次上市） | **高** | **-10% 至 -20% 折价** | +5pct | 部分中小中概 / 当年 DIDI（已私有化）|
| **双重主要上市**（中美 / 中港美） | **中** | **-5% 至 -10% 折价** | +3pct | BABA 9988+BABA / JD 9618+JD / 9866+NIO / 6160+ONC |
| **仅港股 + 红筹结构** | **低** | **0% 至 -3% 折价** | +1pct | 腾讯 0700 / 美团 3690 / 小米 1810 |
| **非中概**（包括外资 / 友邦 / 内地 A 股） | **无** | 0% | 0pct | 茅台 / 工行 / 友邦 |

### 6.3 应用方式

```
modifier_flags = {..., 'pcaob_risk': True}  # 触发条件
# 在 v2 6 维评分输出后：
required_pct_adj = required_pct + pcaob_adj  # 按上表
fair_value_pcaob_adj = fair_value × (1 - pcaob_discount)
```

注意：**PCAOB 折价与 VIE 折价不双重计算**——核心 VIE 中概 ADR（如 BABA）只取较大者，避免叠加。具体取 `max(vie_discount, pcaob_discount)`。

## 九、自检清单（v2 每份报告必填）

报告涉及中概 / AH / 港股 / ADR 公司时，必须完成以下检查：

- [ ] 公司是否在 §二 四分类中明确归类？分类标签是什么？
- [ ] 如为 a_h_dual_listed：是否运行了 §3.3 决策树？输出 A / H / 弃？
- [ ] 如为中概：VIE 分级是什么（§4.2）？折价多少？
- [ ] 如有 ADR：PCAOB 风险等级是什么（§6.2）？是否触发 `required_pct` 加成？
- [ ] VIE 折价与 PCAOB 折价是否取 max 而非叠加？
- [ ] 股息税计算是否使用 §3.2 完整对照表（含港股通自然人 30% 总税）？
- [ ] 是否在报告显著位置标注最终推荐的具体代码（如"推荐 6160.HK 而非 ONC"）？

完成所有勾选 → 报告可发布。任一未通过 → 退回 Stage 1。
