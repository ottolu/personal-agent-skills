# 护城河、商业模式与管理层评估参考（v2）

> 版本：v2.0 — 配合 `V2_SPEC.md` 真理源使用
> 适用：本文为 Skill v2 的护城河 / 商业模式 / 管理层定性分析参考，所有评分必须配合 scorecard.json schema v2 写入。

## 一、设计原则（保留 v1 + v2 强化）

价值投资 80% 的回报来自定性判断，财务只是验证。本文是定性分析的核心。

**v2 关键原则**：
- 护城河 + 商业模式 + 管理层是价值投资最重要的定性环节
- 三个模块**合计占 V2 评分卡 55% 权重**（business_quality 20% + moat 20% + management 15%）
- 评分必须配合 V2_SPEC §四的"6 维评分锚点表"使用
- 任何护城河 / 商业模式 / 管理层维度评分 ≤ 3 触发一票否决（V2_SPEC §五.3）
- v2 引入"否决机制 / 子项独立化 / 双轨评分"三大升级，专门修复 v1 在国企 / 控股集团 / 创始人退出场景的零适配问题

**与 v1 的关键差异**：
- 商业模式六维**新增**行业特殊性提示表（10 类）
- 护城河识别从 7 型扩展到 **11 型**（新增专利 / 监管 / 资本配置者 / 生态）
- 护城河四维评分**新增**"持久性 ≤ 2 整体降级"否决机制
- 护城河扩张性**新增** `expandability_subtype` 字段，消除"无相邻 vs 规模诅咒"二义性
- 管理层 4 维**全部**加入国企 / 控股集团 / 未盈利 biotech / 创始人主导 4 类适配
- 利益绑定维度**三选一**规则，国企央企特别条款
- 控股集团特别规则：**现任 + 继任双轨评分公式**（70% / 30%）
- **新增信息透明度独立子项**，解决拼多多式"持股充足但披露不透明"被一票否决的错误归因

## 二、商业模式六维拆解（v2 加行业适配）

在做任何分析前，必须先回答："这家公司本质上是卖什么的、向谁卖、为什么是它来卖？" 用一句话写出答案。然后展开六维：

### 2.1 通用六维（v1 保留）

| 维度 | 核心问题 |
|---|---|
| **价值主张** | 公司为客户解决了什么问题？没有它客户会怎样？替代成本多高？ |
| **收入结构** | 一次性 vs 经常性？预付 vs 后付？按量/按席/按价值收费？前五大客户占比？地域/产品集中度？ |
| **成本结构** | 固定 vs 可变比例？毛利率构成？哪些成本随规模递减？哪些是刚性的？ |
| **客户结构** | 谁付钱？决策链多长？LTV / CAC？复购率？留存曲线？ |
| **再投资循环** | 每 1 元留存利润能再产生多少利润？增量 ROIC？增长烧资本还是自然长出来？ |
| **关键资源** | 品牌/专利/数据/网络/人才/矿权/渠道？谁拥有？能否复制？ |

### 2.2 行业特殊性提示表（v2 新增）

v1 通用模板对银行 / 保险 / biotech / 地产 / 控股集团 / 周期商品 / 公用事业 / 平台 8 类行业的"六维填空"提示不足。v2 强制每类公司在六维中补对应特殊性：

| company_type | 六维特殊性 |
|---|---|
| `banking` | **成本结构** = 资金成本 + 信用成本 + 业务管理费三层；**客户结构** = 储户 vs 借款人 vs 资管客户三类（同一客户在三个角色之间切换）；**关键资源** = 牌照 + 资本充足率 + 风控体系；**价值主张** = 期限错配 + 风险定价 |
| `insurance` | **关键资源** = 浮存金 / 准备金 / 投资能力三件套；**收入结构** = 承保收益（综合成本率）+ 投资收益（利差/价差）双轮驱动；**价值主张** = 风险转移 + 资金管理；**再投资循环** = 浮存金成本 vs 投资收益率正差 |
| `biotech_unprofitable` | **价值主张** = 未来 FDA 批准 + peak sales NPV，当前无收入；**关键资源** = 管线（pipeline）+ 平台技术 + 临床数据 + 专利集群；**成本结构** = R&D 烧钱率 + SBC + 一次性合作里程碑收入；**客户结构** = 未来病人 / 医保 / KOL / 学会 |
| `pharma_mature` | **关键资源** = 在售药 + 专利剩余年限 + 销售渠道；**收入结构** = 仿制药与原研对决；**成本结构** = R&D 强度 vs 销售费用比；**再投资循环** = patent cliff 时间表是关键 |
| `real_estate` | **收入结构** = 销售（一次性）vs 经营性（持有租赁，经常性）二元；**客户结构** = 个人买家 vs 机构（写字楼 / 商业 / 物流）；**关键资源** = 土储 + 拿地能力 + 融资成本；**再投资循环** = 拿地 → 开发 → 销售 → 回款周期 |
| `holding_conglomerate` | **本质不是一个生意，而是一组生意 + 投资组合**；六维填表必须**对单一业务方向**或采用 SOTP 视角；**关键资源** = 母公司资本配置能力 + 子公司现金流贡献 + 投资组合 NAV；**价值主张** = 永久资本管理 + 跨周期机会捕捉 |
| `cyclical_commodity` | **关键资源** = 矿产 / 资源储量 + 自有物流 + 低成本采矿位置；**收入结构** = 大宗市场定价（无定价权）vs 长协占比；**成本结构** = 单位成本曲线分位是核心；**价值主张** = 资源稀缺 + 提取效率 |
| `utility_infrastructure_reit` | **价值主张** = 政府授予的垄断 + 监管允许回报；**收入结构** = 监管定价（费率管制）；**成本结构** = 高杠杆 + 资本回收期长；**关键资源** = 资产基础 + 牌照 + 网络覆盖 |
| `tech_platform_network` | **关键资源** = 用户网络 + 数据 + 技术 + 开发者生态；**收入结构** = 多重 Take Rate（广告 / 交易 / 订阅 / 增值）；**客户结构** = 双边或多边市场；**再投资循环** = 网络密度 → 用户体验 → 更多用户的复利 |
| `great_company_compounder` | 通用模板适用（这就是巴菲特经典框架的"原型"）；强调**再投资循环**的 ROIC 维持能力 |
| `consumer_brand_premium` | **关键资源** = 品牌 + 渠道 + 心智占有；**价值主张** = 身份认同 / 情感溢价；**成本结构** = 营销与品牌投入是再投资核心 |
| `high_growth_platform` | **收入结构** = GMV + Take Rate 双引擎；**成本结构** = 履约 / 营销 / 补贴的单位经济跟踪；**再投资循环** = 看 contribution margin 转正轨迹 |

**输出**：一页纸商业模式画布 + 一句话总结 + 行业特殊性说明（如适用）。

## 三、护城河十一型识别表（v2：v1 七型 + 新增四型）

v1 的七型经实战检验在以下场景失效：
- 创新药公司（百济）—— "专利集群 + patent cliff 风险"不在七型中
- 国企 / 银行 / 公用事业 —— "监管 / 牌照"未独立成型，归入"无形资产"过于粗糙
- 控股集团（BRK.B）—— "资本配置者本身"是护城河，需单独识别
- 苹果 / 微软 / Adobe —— "生态闭环（软硬+应用商店+开发者）" 超越单一切换成本概念

v2 在 v1 七型基础上**补到十一型**：

| # | 类型 | 识别信号 | 量化指标 | 失效信号 | 代表案例 | v2 新增 |
|---|---|---|---|---|---|---|
| 1 | **无形资产**（品牌/专利/牌照） | 同质产品溢价、监管壁垒、品类心智占有 | 毛利率 vs 同业、品牌溢价率、专利剩余年限 | 提价导致销量下滑、品牌老化、新进入者获批 | 茅台、强生 | |
| 2 | **转换成本** | 客户被迫长期绑定、迁移痛感强 | 客户留存率 > 90%、NRR > 110%、客户生命周期 | 留存率下滑、竞品免费迁移工具、API 开放化 | Salesforce、Oracle、SAP | |
| 3 | **网络效应** | 用户越多越有用、双边/多边市场 | MAU/DAU 增速、跨边密度、Metcalfe 系数、take rate 稳定性 | 区域性碎裂、多归属（multi-homing）、补贴依赖 | Visa、腾讯微信、Meta | |
| 4 | **成本优势** | 单位成本结构性低于同业、长期不可复制 | 单位成本曲线、规模经济斜率、采购议价指数 | 技术变革、原料价格逆转、对手追平 | Costco、福耀玻璃、海螺水泥 | |

### 3.1 多重护城河的识别

最强的公司往往是 **2-3 种护城河叠加**：
- 茅台 = 无形资产（品牌）+ 监管/牌照（生产准入 + 地理保护）+ 高效规模（产能稀缺）
- 苹果 = 差异化（心智）+ 生态（App Store + iCloud）+ 转换成本（数据/账户迁移痛）
- 腾讯 = 网络效应（社交关系链）+ 转换成本（数据/关系绑定）+ 生态（小程序 + 游戏）
- BRK.B = 资本配置者（巴菲特/芒格）+ 监管/牌照（保险）+ 浮存金成本优势
- 招行 = 监管/牌照（银行）+ 转换成本（零售客户粘性）+ 规模经济（网点 + 数字化）

**评分时**：列出所有适用的护城河类型，但综合评分仍按四维（宽度 / 深度 / 持久性 / 扩张性）打分。

### 3.2 护城河类型在 evidence 中的记录方式

scorecard.json 的 `moat.evidence` 字段必须包含：

"moat": {
  "score": 9,

## 四、护城河四维评分（v2 加否决机制 + 子类型）

v1 保留四维：**宽度 / 深度 / 持久性 / 扩张性** 各 1-5 分，合计 4-20 分。

### 4.1 四维定义（v1 保留 + v2 微调）

| 维度 | 含义 | 5 分标准 | 1 分标准 |
|---|---|---|---|
| **宽度** | 多少收入受护城河保护 | 核心业务 80%+ 在护城河内 | < 30% 收入在护城河保护下 |
| **深度** | ROIC - WACC 持续超额多少 | 10 年最差年份 ROIC > WACC + 5pct | ROIC 长期接近或低于 WACC |
| **持久性** | 10 年后还在吗 | 无可见技术/监管/消费习惯威胁 | 5 年内可见明确衰退路径 |
| **扩张性** | 能否延伸到相邻领域 | 已成功 2+ 次品类扩张（如亚马逊从书 → 万物 → 云） | 无相邻赛道或规模诅咒 |

### 4.2 v2 新增：持久性否决规则

**核心问题**（来自神华案例）：v1 公式下"宽度 5 + 深度 5 + 持久性 2 + 扩张性 1" = 13 分（窄护城河），但如果 10 年后护城河不在，宽度 / 深度都是空中楼阁。

v2 引入**持久性否决规则**：

```
if 持久性 ≤ 2:
    综合护城河评级 = 原评级降一档
    # 宽护城河 → 窄护城河
    # 窄护城河 → 无护城河

if 持久性 == 1:
    综合护城河评级 = 原评级降两档
    # 宽护城河 → 无护城河
    # 窄护城河 → 无护城河
```

**理由**：若 10 年后护城河不在，宽度 / 深度 / 扩张性的"当下表现"都失效。神华案例验证—煤炭资源在能源转型下的 10 年视野存疑，因此即使当前宽度 / 深度满分，综合护城河仍要降档。

**实操**：
- 评分时**先打四维分**，**再单独触发否决判定**
- evidence 字段必须显式记录 `durability_veto_applied: true/false` 和触发后的最终评级
- 不要"加权调整"再触发，要保持规则清晰（V2_SPEC §十二命名约定要求）

### 4.3 v2 新增：扩张性二义性消歧（expandability_subtype 字段）

**reviewer 发现的问题**：扩张性 1 分有两种完全不同的含义，v1 不区分会导致评分迷惑：

- **含义 A**："无相邻赛道可扩张"（如神华，资源类天然约束）
- **含义 B**："规模诅咒"（如 BRK.B，公司太大无法找到优质投资标的）
- **含义 C**："市场饱和"（如可口可乐核心碳酸饮料，已覆盖全球）

数值都是 1，但投资含义截然不同：
- 含义 A → 应该接受（生意性质决定，给合理估值即可）
- 含义 B → 接受但调低期待回报率（BRK.B 不再是 10 年 20% 复利）
- 含义 C → 看内生现金返还能力（核心生意稳但要看回购 / 分红）

**v2 新增字段** `expandability_subtype`，枚举值：

```yaml
expandability_subtype:
  - no_adjacent_market      # 无相邻赛道（神华、公用事业）
  - scale_curse             # 规模诅咒（BRK.B 千亿现金流难找标的）
  - market_saturated        # 市场饱和（可口可乐核心碳酸、茅台扩张机会有限）
  - expansion_failed        # 历史扩张失败（如部分多元化失败的传统巨头）
  - active_expansion        # 正在成功扩张（亚马逊云 / Amazon → AWS / Visa → fintech）
```

evidence 字段必须显式说明子类型。

### 4.4 综合护城河评级映射

四维合计得分（先打分，再应用否决规则）：

```
综合分 = 宽度 + 深度 + 持久性 + 扩张性    # 范围 4-20
```

原始评级：
- **宽护城河（wide）**：16-20，巴菲特意义上的"伟大公司"
- **窄护城河（narrow）**：10-15，需要更高安全边际
- **无护城河（none）**：< 10，避免长期持有

**v2 应用否决规则后的最终评级**：

```python
def final_moat_rating(width, depth, durability, expandability):
    raw_score = width + depth + durability + expandability
    if raw_score >= 16:
        rating = "wide"
    elif raw_score >= 10:
        rating = "narrow"
    else:
        rating = "none"

    # v2 持久性否决规则
    if durability == 1:
        # 降两档
        if rating == "wide": rating = "none"
        elif rating == "narrow": rating = "none"
    elif durability == 2:
        # 降一档
        if rating == "wide": rating = "narrow"
        elif rating == "narrow": rating = "none"

    return rating
```

### 4.5 与 scorecard.json moat 6 维分的映射

V2_SPEC §四.3 的 moat 锚点是 0-10 分，与本节四维 4-20 分的映射建议：

| 综合四维分 | 最终评级 | scorecard moat score |
|---|---|---|
| 18-20 | 多重宽护城河持续变宽 | 9-10 |
| 15-17 | 宽护城河 | 7-8 |
| 12-14 | 窄护城河（持续稳定）| 5-6 |
| 10-11 | 窄护城河（收窄中）| 3-4 |
| < 10 | 无护城河 | 0-2 |

**否决规则触发后**：直接按降档后评级取对应分数区间下限。例：神华原始分 16（宽）+ 持久性 2 → 降为窄 → moat score 取 5（窄护城河上限）而非 7。

## 五、管理层评估 4+1 维（v2 重构）

v1 维度：**资本配置 / 坦诚度 / 利益绑定 / 长期主义** 四维。v2 保留四维但每个维度**新增行业适配规则**，并**新增信息透明度独立子项**（5.6）。

### 5.1 资本配置能力（最重要，v2 加行业适配）

#### 通用考察项（v1 保留）
- 过去 10 年留存收益的 ROI
- 历年并购的成功率（是否创造价值 vs 商誉减值）
- 回购时机：是否在低估时买（看回购 PE/PB 中枢）
- 分红连续性 vs 经营现金流稳定性匹配度
- 再投资项目 IRR 是否高于资本成本

#### v2 行业适配

| 公司类型 | 资本配置评估重点 |
|---|---|
| **国企**（神华 / 招行 / 万科 / 工行 / 中石油 / 公用事业等带 `state_owned` 标签） | 默认中性 **5 分**。加分项：高分红 + 回购 + 不盲目扩张。减分项：低 ROIC 大规模 Capex（如部分央企产能扩张项目 IRR < WACC）。**不强求"杰出资本配置者"评分** |
| **控股集团**（BRK.B / Markel / 丹纳赫 / `holding_conglomerate`） | 这是**核心评估维度**，权重最高。9-10 分门槛极高（**巴菲特/芒格级**）。考察：历史 50 年 ROIC、并购质量、回购时机 |
| **未盈利 biotech**（百济 / Moderna / `biotech_unprofitable`） | 不评估"现金流分配"（无 FCF），改评估"**管线投资组合配置**"：临床阶段分散度、平台技术 vs 单品依赖、合作交易（BD）质量、SBC 控制 |
| **高增长平台**（PDD / Sea / Shopify / `high_growth_platform`） | 评估"**用户增长 vs 单位经济**"平衡。看 contribution margin 转正轨迹、获客成本回收期、是否盲目扩张品类 |
| **银行**（招行 / JPM / `banking`） | 评估资本配置 = 信贷资产配置质量 + 资本充足率管理 + 分红率连续性 + 不良核销纪律 |
| **创始人主导**（PDD / 比亚迪 / Tesla / `creator_led`） | 评估创始人个人决策记录 + 长期愿景兑现度，可单独考察"创始人资本配置"维度 |

### 5.2 坦诚度（v2 加行业适配）

#### 通用考察项（v1 保留）
- 致股东信是否讨论失误？还是只报喜不报忧？
- 业绩说明会是否正面回答难题？还是回避？
- 业绩指引达成率（连续多季未达成是危险信号）
- 会计政策是否激进（突然变更、与同业偏离）

#### v2 行业适配

| 公司类型 | 坦诚度评估重点 |
|---|---|
| **国企**（神华 / 招行 / 工行等） | 默认**偏低**（信息披露受外部约束，需经组织部 / 国资委审批）。**不强求 8-10 分**。能在监管要求最低限之上做到主动披露已是加分项 |
| **创始人主导**（拼多多 / 比亚迪 / Tesla） | 看个人公开发言一致性、致股东信质量、是否做"难而正确的事并公开承认"。马斯克 / 黄峥风格差异大，要看 5-10 年发言 vs 兑现比 |
| **控股集团**（BRK.B） | **致股东信是核心**，巴菲特 60 年股东信是标杆。考察是否讨论失败、是否给出年度业绩归因 |
| **银行**（招行 / JPM） | 业绩说明会披露 vs 监管要求最低限差距。是否主动披露不良迁徙、行业敞口、关联交易 |
| **biotech** | 临床数据披露完整性、失败试验是否如实报告、合作交易细节透明度 |

### 5.3 利益绑定（v2 重构：三选一规则 + 国企特别条款）

#### v1 问题（reviewer 反馈）

v1 标准："持股 > 5%" 对国企永远不可能达到（央企高管几乎都是任命制，个人持股 < 0.01%）→ 国企被打 3 分一票否决频发（招行被 PASS、神华被 PASS）。

#### v2 三选一规则

满足任一标准即合格（达到 5 分以上）：

**标准 A：直接持股**
- 核心管理层合计持股 ≥ 1%（**v2 门槛从 5% 降到 1%，更现实**）
- 或 CEO 个人持股 ≥ 0.5%

**标准 B：长期任职 + 经营业绩与薪酬强绑定**
- 核心管理层任职 ≥ 10 年
- 退休前不会跳槽（年龄 + 期权 vesting 锁定）
- 薪酬与长期业绩强挂钩（5 年 ROE / FCF / 股东回报指标）

**标准 C：治理结构良好**
- 控股集团 / 公众公司独董质量高（独立性 + 专业性）
- 关联交易占比 < 5% 且充分披露
- 大股东 vs 小股东利益对齐（如同股同权 + 分红率合理）

#### v2 国企特别条款

| 国企类型 | 利益绑定默认评分 | 触发条件 |
|---|---|---|
| **央企**（神华 / 招行 / 工行 / 中石油 / 中海油 等带 `state_owned` 标签） | 默认 **5 分**（标准 B 满足：高管多为长期任职，业绩与考核挂钩） | 自动适用 |
| **地方国企**（部分省属国资） | 默认 **4 分**（治理结构相对央企略弱） | 自动适用 |
| **国企管理层频繁更换** | 降至 **3 分** | 5 年内 CEO 更换 ≥ 2 次 或 核心管理层平均任期 ≤ 3 年 |
| **国企"治理结构折扣"独立子项** | **-1 分**（央企特有的体制约束） | 自动适用于所有 `state_owned` 标签公司，记入 evidence |

实操示例：
- 招行 = 国企默认 5 分 - 治理折扣 1 分 = **4 分**（合格，不一票否决）
- 神华 = 国企默认 5 分 - 治理折扣 1 分 = **4 分**（合格）
- 某国企 5 年换 3 任 CEO = 3 分 - 治理折扣 1 分 = **2 分**（触发一票否决）

#### v2 创始人 / 民营企业标准

- 创始人主导且持股 ≥ 10%（黄峥、马云、王传福）：**8-9 分起评**
- 创始人持股 < 1%（如部分稀释严重的科技公司）：按通用三选一标准
- 经理人公司（无创始人）：按标准 B / C 评估

### 5.4 长期主义信号（v2 加行业适配）

#### 通用考察项（v1 保留）
- 是否给季度业绩指引（不给是加分项，反映拒绝短期主义）
- 是否在做"难而正确"的事（如逆周期研发投入）
- 研发 / 客户体验投入是否在行业下行时保持
- 是否把过去的失败写进财报和年报

#### v2 行业适配

| 公司类型 | 长期主义评估重点 |
|---|---|
| **国企** | 看 5 年战略规划 / 长期投资连续性 / 政策延续性。能在 5 年规划期保持核心战略不变是加分 |
| **控股集团**（BRK.B） | 看"**永久持有的能力**"（巴菲特"我们最爱的持有期是永远"）。考察重仓股平均持有时间、子公司被出售比率 |
| **高增长平台**（PDD） | 看"**逆周期投入**"。PDD 在同业减投营销时持续高营销加分；Tesla 在低谷期保持 R&D 加分 |
| **biotech** | 看 R&D 强度是否在融资寒冬时保持（甚至加大），管线推进是否按计划 |
| **创始人主导** | 看创始人个人对长期愿景的坚持度（5-10 年公开声明 vs 实际行为一致性） |
| **银行** | 看是否拒绝过度信贷扩张（金融危机前夕的克制是加分项） |

### 5.5 控股集团特别规则：现任 + 继任双轨评分（v2 新增）

#### 背景

V1 在 BRK.B 报告中**手工实现**了"70% × 现任 + 30% × 继任"加权（巴菲特 95 岁，继任问题不可忽视）。V2 将此**标准化**为公式，适用所有 holding_conglomerate / 创始人退休临近 / 大型家族企业。

#### v2 双轨公式

```python
final_management_score = 0.70 × current_score + 0.30 × successor_score
```

**successor_score 评分规则**：

| 继任情况 | successor_score 上限 |
|---|---|
| 继任完全不确定（创始人退休未明 + 多候选人） | 3-4 |
| 继任不明确（已点名但 < 60% 概率） | 6 |
| 继任明确（已确定接班人）但接班人**无业绩** | 7 |
| 继任明确 + 接班人已有 5+ 年管理子公司业绩 | 8-9 |
| 继任明确 + 接班人是另一位巴菲特级人物（理论上限） | 10 |

#### 适用范围

- `holding_conglomerate` 全员
- 创始人年龄 ≥ 70 岁 且 公司高度依赖创始人决策
- 家族企业第二代 / 第三代接班期
- 任何 modifier_flags 含 `creator_led` 且创始人 ≥ 65 岁

#### 实操示例

**BRK.B 案例**：
- current_score = 10（巴菲特/芒格，资本配置 10 + 坦诚度 10 + 利益绑定 9 + 长期主义 10）
- successor_score = 6（Greg Abel 已点名但运营 vs 投资能力不完全等价）
- final_management_score = 0.70 × 10 + 0.30 × 6 = **8.8**

**对比 v1**：手工填了 9.0 看似合理但缺乏公式约束，新接班人若降级到 5（continuum），v2 公式自动给出 8.5，v1 没有这种灵敏度。

### 5.6 信息透明度独立子项（v2 新增）

#### 背景

V1 reviewer 反馈：拼多多 management = 3 触发一票否决，但黄峥实际"持股 27% + 公司执行力极强 + 长期资本配置优秀"。错误归因为"管理层质量差"，实际是"信息披露不透明"——这两个是**完全不同的问题**。

V1 把两个问题揉在一起评分，导致：
- 误判：高质量但低透明 → 被一票否决
- 漏判：质量平平但披露详尽 → 不会被一票否决

#### v2 拆分

**管理层最终评分 = 加权（资本配置 + 坦诚度 + 利益绑定 + 长期主义）+ 信息透明度独立子项**

| 子项 | 评分范围 | 计算方式 |
|---|---|---|
| 资本配置 | 0-10 | 见 5.1 |
| 坦诚度 | 0-10 | 见 5.2（与透明度的区别：坦诚度看"愿不愿意承认错误"，透明度看"客观披露信息的程度"） |
| 利益绑定 | 0-10 | 见 5.3 |
| 长期主义 | 0-10 | 见 5.4 |
| **信息透明度**（v2 新增） | 0-10 | 财报详细度 / 业绩说明会质量 / 关联交易披露 / 业务分部披露 / 非财务指标披露 |

#### 加权公式

```python
core_management_score = (capital_alloc + candor + alignment + long_term) / 4
transparency_score = 0-10 单独评分

# 加权方式（任选其一，scorecard.json 应明确标注）：

# 方式 A：简单平均（适用 5 维平等重要）
final = (core_management_score × 4 + transparency_score) / 5

# 方式 B：核心 80% + 透明度 20%（推荐，适用大多数公司）
final = 0.8 × core_management_score + 0.2 × transparency_score
```

#### 拼多多案例（v2 修正）

V1：management = 3 → 一票否决（PASS）
V2：
- 资本配置 = 7（黄峥+陈磊，电商业务执行力强 + Temu 海外扩张资本配置审慎）
- 坦诚度 = 4（业绩说明会少、致股东信少）
- 利益绑定 = 9（黄峥持股 27%，标准 A 满足）
- 长期主义 = 8（逆周期投入营销 + 海外扩张）
- 信息透明度 = 3（财报披露最低限 + 业务分部不细分）
- core = (7+4+9+8)/4 = 7.0
- final = 0.8 × 7.0 + 0.2 × 3 = **6.2**（合格，不一票否决）

**V2 修正效果**：拼多多从一票否决 PASS → 进入"watchlist_quality_only"（质量过关但需评估估值）。这是 V1 → V2 的关键修复之一。

## 八、红旗清单（管理层维度）

以下红旗触发后须在 scorecard.json `red_flags.critical_flags` 中记录：

## 九、本模块输出要求（v2 强化）

完整分析必须产出三份结构化结论：

### 9.1 商业模式一页纸
- 通用六维填空
- 一句话总结："这家公司本质上是卖 X 给 Y，靠 Z 不可复制"
- 行业特殊性提示（按 §2.2 表，如适用）
- 输出位置：报告正文 + `scorecard.json` 的 business_quality.evidence

### 9.2 护城河四维评分卡
- 适用护城河类型（§三十一型中的 1 个或多个）
- 四维分（宽度 / 深度 / 持久性 / 扩张性 各 1-5）
- `expandability_subtype` 字段填值（§4.3）
- 持久性否决判定（§4.2）：是否触发降档、降档后最终评级
- 映射到 scorecard.json 的 moat.score（0-10）
- evidence 字段必须包含：moat_types + 四维分 + expandability_subtype + durability_veto_applied

### 9.3 管理层多维评分
- 资本配置 / 坦诚度 / 利益绑定 / 长期主义 4 子项分（各 0-10）
- 信息透明度独立子项分（v2 新增，0-10）
- 国企特别条款是否触发 / 适用
- 控股集团双轨评分公式是否适用（§5.5）：如适用，需 current_score + successor_score 双值
- 加权公式选择（A 或 B，见 §5.6）
- 映射到 scorecard.json 的 management.score（0-10）
- evidence 字段必须包含：4 + 1 子项分 + 继任评估如适用 + 行业适配规则触发

## 十、与 v1 关键差异速查表

| 维度 | v1 | v2 |
|---|---|---|
| 商业模式六维 | 通用模板 | 通用 + 10 类行业特殊性提示 |
| 护城河型数 | 7 型 | **11 型**（新增专利 / 监管 / 资本配置者 / 生态） |
| 护城河四维评分 | 简单求和 | 求和 + **持久性 ≤ 2 整体降级**否决 |
| 扩张性歧义 | 含糊 | **expandability_subtype** 字段消歧（5 类） |
| 管理层维度数 | 4 维 | **4 + 1 维**（新增信息透明度独立子项） |
| 利益绑定标准 | 持股 > 5% | **三选一**（持股 / 长任职+绑定 / 治理结构）+ 国企特别条款 |
| 控股集团评分 | 通用打分 | **双轨评分公式** 70% 现任 + 30% 继任 |
| 国企适配 | 无 | **管理层 4 维全部适配** + 央企治理折扣 -1 |
| 创始人退出场景 | 不适配 | 双轨评分 + 长期主义看创始人个人 |
| 公司类型数 | 6 种 | **18 类 company_type**（V2_SPEC §二锁定枚举） |
| 一票否决误伤 | 拼多多 = 3 被 PASS | **拆分透明度独立项** → 拼多多 = 6.2 可保留 |
