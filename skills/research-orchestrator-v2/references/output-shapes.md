# Output shapes

Pick the shape that best matches downstream use.

## 1. Exploration output
- what this space is
- taxonomy / landscape map
- key players or approaches
- major fault lines
- open questions
- recommended next research lanes

## 2. Decision-support output
- recommendation up front
- comparison matrix
- key tradeoffs
- risks / caveats
- evidence highlights
- suggested action path

## 3. Executive-brief output
- 3-7 key judgments
- concise evidence bullets
- biggest uncertainty
- implications for the audience
- next actions

## 4. Technical-due-diligence output
- architecture breakdown
- evidence quality and benchmark scrutiny
- implementation realities
- technical risks / unknowns
- final technical judgment

## Reusable blocks

Use these blocks when helpful:
- one-paragraph executive summary
- key questions
- evidence table
- comparison table / dimension matrix
- disagreement tracker
- confidence levels
- recommendation / next steps

## Style rules

- optimize for usefulness, not maximum length
- compress aggressively for executive audiences
- avoid raw note dumps unless the user explicitly asked for notes
- if uncertainty is high, say so plainly instead of hiding it behind verbose prose
