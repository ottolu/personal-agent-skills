# Multi-agent topologies

## Full Deep Research default topology

For `research-orchestrator`, this is the **default minimum topology**, not an optional suggestion.

1. Planner
   - define the research frame
   - split the work into dimensions
   - set success criteria and unknowns
   - identify what must be verified vs what is background

2. Primary-source scout
   - collect Tier 1 sources
   - build a source map
   - identify canonical names, dates, entities, and artifacts
   - normalize terminology before deeper research begins

3. Domain researcher A
   - investigate the first major dimension
   - examples: architecture / timeline / market structure / product map

4. Domain researcher B
   - investigate the second major dimension
   - examples: benchmarks / GTM / sector implications / ecosystem / adoption

5. Skeptic / red-team
   - look for contradictions, adverse evidence, overclaiming, outdated claims, weak sourcing, and missing baselines
   - pressure-test the main thesis rather than merely summarizing alternatives

6. Main synthesizer
   - merge, de-duplicate, classify evidence, and write the final view
   - separate facts, inferences, and judgments

## When to expand beyond the minimum

Add more lanes when:
- the topic naturally splits into 3+ distinct research dimensions
- breadth matters as much as depth
- there are multiple stakeholder lenses to cover
- verification needs are unusually high
- the user explicitly asks for broad, deep, or multi-angle research

## Default split patterns

### Market / macro / investing topic
- market facts and timeline
- driver analysis and transmission mechanisms
- sector / style / asset-chain implications
- red-team

### Technical topic
- architecture / implementation
- benchmarks / eval quality
- ecosystem / tooling / adoption
- red-team

### Product / company topic
- product / capability map
- market / GTM / adoption
- competition / positioning
- red-team

### Landscape topic
- taxonomy / segmentation
- top players and positioning
- evidence by segment
- strategic implications
- red-team

## Failure modes to avoid

- silent downgrade to single-thread work while presenting the output as deep research
- multiple agents searching the same thing with no role distinction
- no primary-source lane
- no contradiction-finding / red-team pass
- a final memo that summarizes but does not judge
- collapsing facts, inferences, and opinions into one bucket
- using long output length as a substitute for research depth
- producing a "red-team" section that is only cosmetic and did not come from an actual skepticism lane
