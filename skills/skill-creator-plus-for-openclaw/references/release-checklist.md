# Release checklist

Before packaging a skill:

- `SKILL.md` frontmatter is valid
- name and folder name match intent
- description clearly states what the skill does and when to use it
- no placeholder TODO text remains
- references are actually useful and not dead weight
- scripts exist only if they provide repeatable leverage
- the skill has at least a small realistic eval set or test plan if the task is important
- anti-patterns have been reviewed
- packaging output path is clear

After packaging:
- summarize what changed
- note known limitations
- propose the next iteration if the skill is still immature
